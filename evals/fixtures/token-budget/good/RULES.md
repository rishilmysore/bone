# Rules

Always-loaded conventions for this micro-repo. Budget: 600 tokens.

- Start by reading STATE, then resume its next action.
- One atomic commit per task, imperative mood.
- A bugfix lands its failing test first, then the fix.
- Docs describe the present; history lives in git.
- Verify before claiming done: run the fast gate and paste output.
- Blocked twice on the same symptom: stop and write the blocker down.
- Comments exist only for constraints the code cannot express.
- New dependency: verify exact name and current version in its registry first.
